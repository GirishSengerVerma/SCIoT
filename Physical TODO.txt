Sensors:
	Announce on startup of script:
		TOPIC sensors/metadata/<location>/<measure>
		MESSAGE {
			instanceId,
			name,
			location,	// STUTTGART_MAX_EYTH_SEE, STUTTGART_KILLESBERG_PARK
			measure,	// LIGHT, WATER_LEVEL
		}

	Periodically send:
		TOPIC sensors/telemetry/<location>/<measure>
		MESSAGE {
			instanceId,
			timestamp, 	// Current ISO Datetime String!! e.g. 2022-07-16T09:59:42.823015+00:00
			unit,		// BOOLEAN, METERS
			value		// true/false, decimal
		}

Actuators:
	Announce on startup of script:
		TOPIC actuators/metadata/<location>/<type>
		MESSAGE {
			instanceId,
			name,
			location,	// STUTTGART_MAX_EYTH_SEE, STUTTGART_KILLESBERG_PARK
			type,		// ALARM_LIGHT, ALARM_SOUND, WATER_PROTECTION_WALL
		}

	Listen to status messages and update correspondingly:
		TOPIC actuators/status/<location>/<type> 
		MESSAGE {
			instanceId,
			timestamp,	// Current ISO Datetime String!! e.g. 2022-07-16T09:59:42.823015+00:00
			enabled,	// true/false
		}
